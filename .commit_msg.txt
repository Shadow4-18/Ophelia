Add DeepSeek provider + transient-error fallback chain

DeepSeek is OpenAI-compatible and very cheap (V4 Flash ~$0.14/1M input),
making it ideal for cost-sensitive background roles and as a backup.

- DeepSeek as a first-class provider: DEEPSEEK_API_KEY/BASE_URL/MODEL plus
  per-role model env vars (DEEPSEEK_*_MODEL). Builds as an
  OpenAICompatibleBackend; included in auto-pick and the setup wizard's
  provider menu and per-role menu.
- Fallback system: OPHELIA_FALLBACK_PROVIDERS (ordered) +
  OPHELIA_FALLBACK_MODEL. When the primary hits a transient error (429, 5xx,
  network, timeout) the agent retries each fallback in order before failing.
  Non-transient errors (400/401) surface immediately so retries aren't
  wasted. Wired into AgentLoop._complete via _call_with_fallback, and into
  the curator and dream loops via a shared providers/fallback.py helper.
- Media roles now auto-pick when the primary can't serve them (e.g. DeepSeek
  has no image/video gen) instead of inheriting an incapable primary.
- Docs: README and config.example.env updated for DeepSeek + fallback.
