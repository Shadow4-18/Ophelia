Add pluggable web search backends (Tavily/Serper/Brave) + setup option

Grok has built-in live search, but DeepSeek/OpenAI/Ollama do not — so on
those providers Ophelia relies on the web_search tool. The old DuckDuckGo
scraper is free but unreliable (often returns nothing). Add real keyed
backends for reliable results.

- OPHELIA_WEB_SEARCH_PROVIDER: auto | duckduckgo | tavily | serper | brave
  (default auto = first API key set, else duckduckgo).
- TAVILY_API_KEY / SERPER_API_KEY / BRAVE_API_KEY for the keyed backends.
- web_search.py dispatches to the configured backend and falls back to
  DuckDuckGo on any failure/empty result, so a search never hard-fails.
- Setup wizard: new "Web search" menu item with provider picker + key prompts.
- Tool description updated to nudge models without built-in search to use it.
- Docs: README and config.example.env document the new backends.
